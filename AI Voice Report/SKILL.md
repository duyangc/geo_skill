# AI 搜索声量报告生成技能

## 快速工作流

### 【前置必做】品牌深度调研

**在生成报告前，必须通过互联网搜索或向大模型提问，清晰了解目标品牌。** 调研顺序和方法如下：

#### 第一层：官方权威来源（优先）
1. **百度百科**：搜索 `baike.baidu.com/item/{品牌名}`，获取品牌定位、成立背景、核心业务
2. **品牌官网**：直接访问官网首页及 `/about`、`/products` 等子页面，获取产品线、解决方案、目标客户
3. **天眼查 / 企查查**：获取工商注册信息、主营业务描述、股东背景

#### 第二层：降级检索（当第一层受反爬或词条缺失时）
4. **行业媒体与新闻通稿**：搜索 `"{品牌名}" 发布 产品 解决方案 行业`，从 36kr、钛媒体、IT之家、品牌公众号等获取信息
5. **垂直行业平台**：结合品牌所在行业（金融/医疗/零售等）搜索相关报道

#### 第三层：网络基础阵地排查（基于 GEO/SEO 原理）

排查以下5项，每项记录状态（正常 / 受限 / 缺失），作为报告第2章和第4章的诊断依据：

| 排查项 | 方法 | GEO 意义 |
|--------|------|----------|
| 官网访问连通性 | 直接访问，测试是否正常加载 | 官网是 AI 引用的首要权威来源（E-E-A-T 中的 Authoritativeness） |
| 百科词条状态 | 检查百度百科词条是否存在及内容完整度 | 百科是结构化实体知识的核心载体，缺失则 AI 无法建立品牌实体认知 |
| 媒体报道稀缺度 | 搜索品牌名，判断高质量第三方报道数量 | 第三方引用是 E-E-A-T 中 Trustworthiness 的核心信号 |
| 搜索结果质量 | 观察搜索结果前10条：是否被官网/权威平台主导 | 搜索结果结构直接影响 AI 爬取优先级 |
| 社交平台内容密度 | 检查微信公众号、微博、知乎等是否有品牌内容沉淀 | 多平台内容分发是 GEO 内容覆盖广度的基础 |

#### 调研完成标准
调研结束时，你应当能明确回答：
- ✅ 品牌的核心定位是什么（一句话）
- ✅ 主要产品线有哪些（2–5个）
- ✅ 目标客户群是谁
- ✅ 主要竞品是哪几家
- ✅ 网络基础阵地5项的状态

**若以上信息仍不完整，应在生成报告前向用户补充询问，不得凭空推断。**

1. **收集品牌信息**（结合调研结果，不足时向用户询问，数据必须真实，不得杜撰）：
   - 品牌名称（**严格使用用户提供的原始名称，不得自行修改或简化**）
   - 业务描述/标语
   - 报告期（年月）
   - 各 AI 平台测试数据（分数、提及率、特征描述）
   - 竞品名称和数据
   - 核心关键词及提及率

2. **读取规范**：查阅 `references/report-structure.md` 了解章节结构和配色规则

3. **复制模板**：将 `assets/template.html` 作为起点，将 `{{占位符}}` 替换为实际数据

4. **输出文件**：命名为 `{品牌简称}_AI声量报告.html`，保存到用户当前工作目录

5. **【输出后质检】**：

   生成报告初稿后，必须对照以下8个维度逐项检查，每项标注 ✅ Pass 或 ❌ Fail，并写出简短判定理由。

   **质检维度：**
   ① **数据真实性**：报告中所有数字（平台分数、提及率等）是否有实测或真实调研来源，不得虚构
   ② **竞品分析深度**：竞品对比是否触及本质差异（技术路线/市场定位/客户结构），而非表面功能罗列
   ③ **建议业务匹配度**：GEO 优化建议是否结合品牌具体业务场景、产品线和客户群，而非通用模板
   ④ **品牌名称一致性**：全文统一使用用户原始品牌名称，无缩写/替换/简化
   ⑤ **章节格式合规性**：基础版仅含前5章，第4章为竞品对比、第5章为问题诊断（不可颠倒）；第1章无 KPI 汇总卡片；Footer 两行格式正确；配色符合 #D92B2B；§6任务列表使用纵向堆叠布局；§6三阶段各5条任务数量大致相等
   ⑥ **内容逻辑自洽性**：第4章(竞品差距)的结论为第5章(问题诊断)提供输入，第5章建议与第3章诊断有因果关系
   ⑦ **GEO/SEO 专业深度**：引用具体原理（E-E-A-T/实体知识图谱/AI 可引用性等），而非仅通俗表述
   ⑧ **可读性与视觉完整性**：无样式破损、无 `{{xxx}}` 占位符残留、图表数据完整；§6中Phase 03的HTML结构完整（`</ul>`闭合、`<div class="phase-kpi">`存在、KPI项不缺失）

   **判定流程：**
   - 全部 ✅ → 报告通过，附质检通过声明（"已通过8项质检：①数据真实性 ②竞品分析深度 ③建议业务匹配度 ④品牌名称一致性 ⑤章节格式合规性 ⑥内容逻辑自洽性 ⑦GEO/SEO专业深度 ⑧可读性与视觉完整性"），完成交付
   - 存在任何 ❌ → 按 Fail 条目逐一修正 → 重新逐项自检 → 直到 8/8 全部 ✅ 才最终交付

## 关键约束

- **品牌名称不可修改**：报告中所有位置（标题、卡片、表格、Footer）必须使用用户提供的原始品牌名称，不得缩写、翻译、替换或自行简化
- **第1章禁止**添加"AI综合声量得分"、"AI正向提及率"、"品牌空白提及率"三个 KPI 汇总卡片
- **基础版只生成前5章**；完整版可选增加§3.5品牌被动提及矩阵和§6发现的问题与GEO优化行动路径
- **配色固定为红色系**：Header 渐变 `#1A1A2E → #D92B2B`，主品牌色 `#D92B2B`
- 所有图表用纯 HTML+CSS 实现，不引入外部 JS 图表库，不使用Emoji
- 目标品牌在竞品对比表中使用 `class="target-row"` 高亮
- **Footer 固定格式**（不可更改文字，仅替换 `{{REPORT_YEAR_MONTH}}`）：
  - 第一行：`报告仅供参考，数据基于{{REPORT_YEAR_MONTH}}公开信息生成`（`{{REPORT_YEAR_MONTH}}` 替换为系统当前年月，格式如 `2026年3月`）
  - 第二行：`本报告由中关村科金GEO分析框架生成`

## 报告生成注意事项（基于实际项目经验）

> 以下规则来自多次报告迭代中暴露的问题，生成前必须逐条确认。

### 章节结构顺序

- **第4章为竞品对比，第5章为问题诊断**：先展示竞争格局（让客户看到差距），再给出问题诊断（针对差距提出对策），逻辑更顺畅。严禁颠倒顺序。
- **§3.5品牌被动提及矩阵**：位于第3章末尾、第4章(竞品对比)之前。
- **§6行动路径**：位于第5章(问题诊断)之后、Footer之前。

### 措辞与承诺控制

- **禁止具体时间承诺**：任务描述中不得出现"30天内""90天内"等具体时间限定，改为"大幅提升""显著改善""有效降低"等方向性表述。
- **禁止具体数字目标**：任务描述中不得出现"提及率从13%提升至50%""从0到30%"等具体数字目标。数字目标只出现在阶段底部的"阶段GEO目标指标"（KPI）中，不出现在每条任务的描述里。
- **KPI指标可验证原则**：KPI中使用的指标必须能被客户直接通过搜索测试验证，不得使用"AI综合声量得分""认知覆盖率"等没有统一计算标准的自创指标。应使用"某关键词搜索下品牌是否被主动推荐""是否进入推荐前三顺位""某平台某场景中是否被提及"等可直接搜索确认的表述。
- **KPI双层结构**：每个KPI指标必须包含两层信息——"达到XX目标，从而解决XX问题"。例如："通用推荐场景宝马提及率从55%提升至70%以上——解决泛品类搜索中被AI遗忘的致命问题"。
- **任务描述中的"目标："统一改为"预期效果："**：每条任务末尾的产出目标不使用"目标：30天内..."的表述，使用"预期效果：大幅提升..."。
- **不写具体媒体/KOL名称**：任务描述中不得出现"懂车帝""汽车之家""36氪""知乎""钛媒体"等具体媒体平台名称，统一使用"主流汽车媒体""问答社区""科技财经媒体""内容平台"等概括性表述。写具体名称会导致客户期望与实际执行之间的落差。

### 三阶段行动路径规则（§6）

- **三阶段任务数须均衡**：每个阶段的行动项数量应大致相等（建议各5条左右），三阶段内容量不可差距过大，否则视觉上长短不一。
- **竖向堆叠，非横排三列**：roadmap-grid使用 `display: flex; flex-direction: column` 纵向排列，不使用 `grid-template-columns: repeat(3, 1fr)` 横向三列。每条任务较长时竖向排列阅读体验更好。
- **去除内部运营类任务**：三阶段中只写客户可感知的交付型任务（如内容撰写、口碑素材征集、媒体发布等）。服务商内部工作（如监测系统搭建、基线数据面板、预警机制、内部复盘会等）不写入报告。
- **KPI标题统一为"阶段GEO目标指标"**：不使用"阶段GEO目标指标（第X个月末验收）"带具体时间的形式。
- **HTML结构完整性检查**：生成§6后必须检查每个阶段的HTML结构是否完整——特别检查Phase 03是否有正确的`</ul>`闭合、`<div class="phase-kpi">`标题和KPI项是否齐全。

### 数据统计口径

- **提及率仅计算通用场景题**：统计提及率时，必须将测试题分为两类——含品牌关键词的"品牌专属题"和不含品牌关键词的"通用推荐题"。提及率仅以通用题为分母计算，品牌专属题不计入分母。例如：90题中有33题含品牌名、57题不含，则提及率=57题中品牌被提及的题数/57。
- **竞品对比需覆盖全量**：如果用户提供了包含竞品列表的知识库，报告中必须覆盖全部指名竞品，不得仅选取少数代表性品牌进行分析。竞品的AI提及数据需基于测试结果逐题统计。

### 措辞准确性

- **两平台结论必须分别表述**：严禁将两个AI平台的结论笼统合并为"两大平台均XXX"。当A平台表现优于B平台时，必须分别阐述各自表现后，再总结差异。例如：不能先说"豆包回答较为概括、缺少量化数据"，然后结论又说"两大平台已达专家级"——前后矛盾。
- **禁止使用"100%正面""专家级"等过度绝对化表述**：品牌认知评价按平台分别评估。表现好的平台可评价为"接近专家级"/"深度较高"，表现一般的平台应为"回答充分但缺乏深度"/"准确但概括"。

### 平台展示规则

- **仅展示有实测数据的平台**：第1章的评分卡片网格中，只展示实际进行了测试的AI平台。如果仅测试了2个平台，使用2列布局（platform-grid使用grid-template-columns: 1fr 1fr），不得添加其他平台的估算卡片或评分。

### 三阶段行动路径规则（§6）

- **时间格式**：三阶段时间统一使用"第X-X个月"格式（如"第1-3个月"），不得使用具体年份/月份（如"2026年6月"）。
- **阶段目标措辞**：阶段目标统一使用"100%覆盖"语义，而非百分比递增式表述。
  - Phase 01："确保优化后所有通用推荐场景中品牌均被AI提及，实现100%覆盖"
  - Phase 02："维持100%覆盖，建立差异化品类认知"
  - Phase 03："维持100%覆盖并扩展至更多AI平台"
- **行动项不锁定具体数量**：行动指南中的文章/内容发布不得出现具体篇章数（如"发布5篇"），使用"发布行业场景深度方案内容"等开放式表述。

### CSS / 样式约束

- **横条图标签格式**：左侧标签格式为"品牌名称 + 空格 + 指标名称"，标签区域宽度≥200px，使用 white-space: nowrap 防止截断换行。
- **表格样式**：query-table的th需加white-space:nowrap防列头换行；td使用vertical-align:middle实现垂直居中。comp-table适配多列（多达12列）时th/td字号用12px。
- **opp-card字体一致性**：.opp-card CSS类必须显式设置font-size:13px（与info-box、warn-box保持一致），否则<strong>标签内的文字将继承浏览器默认字号导致同一卡片内字体大小不一致。

## 章节结构速查

| 章节 | 核心组件 | 适用场景 |
|------|---------|---------|
| 第1章 | 6个平台卡片（环形得分）+ insight-box 结论 | 必选 |
| 第2章 | 认知现状卡（已知/空白两栏）+ 关键词提及率横条图 | 必选 |
| 第3章 | 5个类别卡片（测试问题+AI特征+结论） | 必选 |
| §3.5 | 品牌被动提及矩阵（query-table + mention状态） | 非品牌类必选，品牌类可选 |
| 第4章 | P0/P1/P2 问题卡片 + insight-box 总结 | 必选 |
| 第5章 | 竞品对比表 + 多维横条图 + 差距分析 | 必选 |
| §6 | 发现总结 + 三阶段行动路径(roadmap-grid) + 优先级矩阵(geo-table) + 行动总纲(insight-box) | 完整版可选（非品牌类建议包含） |

### §6 三阶段行动路径任务列表规范

- **时间格式**：统一使用"第X-X个月"格式（如"第1-3个月"），不得使用具体年份月份
- **阶段目标用语**：使用"100%覆盖"语义（如"确保100%覆盖""维持100%覆盖"），而非百分比递增式
- **行动项措辞**：动作标题中不得出现具体篇章数（如"发布5篇"应为"发布行业场景内容"）

**必须使用纵向堆叠布局**，禁止双栏 `——` 分隔写法。每项结构：

```html
<li class="task-item">
  <div class="task-title">动作标题（加粗）</div>
  <div class="task-desc">详细描述（灰色缩进 + 左侧竖线装饰）</div>
</li>
```

**阶段命名原则**：
- 品牌类报告：止血修复期 → 筑底提升期 → 升维领跑期
- 非品牌类报告：紧急补位期 → 认知强化期 → 长效运营期

## 平台评分颜色规则

| 分数 | SVG stroke | tag class | 标签文字示例 |
|------|-----------|-----------|-------------|
| ≥ 60 | `#10B981` | `tag-ok` | 高潜力 |
| 40–59 | `#F59E0B` | `tag-mid` | 中等机会 |
| < 40 | `#D92B2B` | `tag-weak` | 空白市场 |

`stroke-dashoffset` 公式：`round(213.63 × (1 - score/100))`
